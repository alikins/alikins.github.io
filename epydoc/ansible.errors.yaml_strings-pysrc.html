 41  YAML_COMMON_DICT_ERROR = """\ 
 42  This one looks easy to fix.  YAML thought it was looking for the start of a 
 43  hash/dictionary and was confused to see a second "{".  Most likely this was 
 44  meant to be an ansible template evaluation instead, so we have to give the 
 45  parser a small hint that we wanted a string instead. The solution here is to 
 46  just quote the entire value. 
 47   
 48  For instance, if the original line was: 
 49   
 50      app_path: {{ base_path }}/foo 
 51   
 52  It should be written as: 
 53   
 54      app_path: "{{ base_path }}/foo" 
 55  """ 
 56   
 57  YAML_COMMON_UNQUOTED_VARIABLE_ERROR = """\ 
 58  We could be wrong, but this one looks like it might be an issue with 
 59  missing quotes.  Always quote template expression brackets when they 
 60  start a value. For instance: 
 61   
 62      with_items: 
 63        - {{ foo }} 
 64   
 65  Should be written as: 
 66   
 67      with_items: 
 68        - "{{ foo }}" 
 69  """ 
 70   
 71  YAML_COMMON_UNQUOTED_COLON_ERROR = """\ 
 72  This one looks easy to fix.  There seems to be an extra unquoted colon in the line 
 73  and this is confusing the parser. It was only expecting to find one free 
 74  colon. The solution is just add some quotes around the colon, or quote the 
 75  entire line after the first colon. 
 76   
 77  For instance, if the original line was: 
 78   
 79      copy: src=file.txt dest=/path/filename:with_colon.txt 
 80   
 81  It can be written as: 
 82   
 83      copy: src=file.txt dest='/path/filename:with_colon.txt' 
 84   
 85  Or: 
 86   
 87      copy: 'src=file.txt dest=/path/filename:with_colon.txt' 
 88  """ 
 89   
 90  YAML_COMMON_PARTIALLY_QUOTED_LINE_ERROR = """\ 
 91  This one looks easy to fix.  It seems that there is a value started 
 92  with a quote, and the YAML parser is expecting to see the line ended 
 93  with the same kind of quote.  For instance: 
 94   
 95      when: "ok" in result.stdout 
 96   
 97  Could be written as: 
 98   
 99     when: '"ok" in result.stdout' 
100   
101  Or equivalently: 
102   
103     when: "'ok' in result.stdout" 
104  """ 
105   
106  YAML_COMMON_UNBALANCED_QUOTES_ERROR = """\ 
107  We could be wrong, but this one looks like it might be an issue with 
108  unbalanced quotes.  If starting a value with a quote, make sure the 
109  line ends with the same set of quotes.  For instance this arbitrary 
110  example: 
111   
112      foo: "bad" "wolf" 
113   
114  Could be written as: 
115   
116      foo: '"bad" "wolf"' 
117  """
