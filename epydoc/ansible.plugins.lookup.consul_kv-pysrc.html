 20  ''' 
 21  Lookup plugin to grab metadata from a consul key value store. 
 22  ============================================================ 
 23   
 24  Plugin will lookup metadata for a playbook from the key value store in a 
 25  consul cluster. Values can be easily set in the kv store with simple rest 
 26  commands e.g. 
 27   
 28  curl -X PUT -d 'some-value' http://localhost:8500/v1/kv/ansible/somedata 
 29   
 30  this can then be looked up in a playbook as follows 
 31   
 32  - debug: msg='key contains {{item}}' 
 33    with_consul_kv: 
 34      - 'key/to/retrieve' 
 35   
 36   
 37  Parameters can be provided after the key be more specific about what to retrieve e.g. 
 38   
 39  - debug: msg='key contains {{item}}' 
 40    with_consul_kv: 
 41      - 'key/to recurse=true token=E6C060A9-26FB-407A-B83E-12DDAFCB4D98')}}' 
 42   
 43  recurse: if true, will retrieve all the values that have the given key as prefix 
 44  index: if the key has a value with the specified index then this is returned 
 45         allowing access to historical values. 
 46  token: acl token to allow access to restricted values. 
 47   
 48  By default this will lookup keys via the consul agent running on http://localhost:8500 
 49  this can be changed by setting the env variable 'ANSIBLE_CONSUL_URL' to point to the url 
 50  of the kv store you'd like to use. 
 51   
 52  '''
