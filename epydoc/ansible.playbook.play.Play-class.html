A play is a language feature that represents a list of roles and/or
task/handler blocks to execute on a given set of hosts.

Usage:

   Play.load(datastructure) -> Play
   Play.something(...)

_compile_roles(self)

source code 

Handles the role compilation step, returning a flat list of tasks with the lowest level dependencies first. For example, if a role R has a dependency D1, which also has a dependency D2, the tasks from D2 are merged first, followed by D1, and lastly by the tasks from the parent role R last. This is done for all roles in the Play.
